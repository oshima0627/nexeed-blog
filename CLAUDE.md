# NEXEED BLOG — Claude 作業ルール

このファイルはClaude Codeが常に参照するルール定義ファイルです。
全ての作業において以下のルールを遵守してください。

---

## 時事ネタ記事の作成ルール（最重要）

### 1. 情報源の信頼性確保
- **必ず公式サイト・信頼性の高いメディアから最新情報を取得すること**
  - 政府・省庁の公式サイト（首相官邸、経産省、厚労省、内閣府など）
  - 主要経済紙（日本経済新聞、Bloomberg、Reuters）
  - 大手シンクタンク（第一生命経済研究所、大和総研、みずほリサーチなど）
  - 国際機関（IMF、OECD、世界銀行など）
  - 企業公式発表・プレスリリース

### 2. 情報の精査・ファクトチェック
- **WebSearchとWebFetchツールを使って実際の情報を取得・確認すること**
- 複数の信頼できるソースで裏付けが取れた情報のみ記載する
- 1つのソースしか確認できなかった情報は「～との報道」「～によると」と明示する
- 数値データ（統計、金額、割合）は必ず出典を明記すること
- 確認できなかった情報は記事に含めない

### 3. 禁止事項
- **AIが生成した推測・予測を事実として記載することを禁止**
- **出典不明の数値・統計を使用することを禁止**
- **公式確認できていない情報を断定的に記載することを禁止**
- 誇張表現・センセーショナルな見出しを避ける

### 4. 記事作成フロー（必須）
1. WebSearchで最新ニュースを検索（複数クエリで複合的に調査）
2. 公式サイト・信頼できるメディアのURLをWebFetchで取得・確認
3. 複数ソースで情報の整合性を確認
4. 確認された事実のみで記事を構成
5. 記事末尾または本文中に出典URLを明記

### 5. 記事フォーマット
- カテゴリー：投資 / 子育て / ITエンジニア / 副業 のいずれか
- フロントマターに `verified: true` を追加（情報精査済みの証明）
- フロントマターに `sources` フィールドで主要出典を列挙

---

## 記事全般のルール

- Markdownフロントマターのフォーマットを遵守する
- 日付は記事作成時の実際の日付を使用する
- **`date` フィールドには記事ファイルの実際の作成日時を ISO 8601 形式（`"YYYY-MM-DDTHH:MM:SS"`）で記載すること**（推測・固定値の使用禁止）
- カテゴリーは「投資」「子育て」「ITエンジニア」「副業」の4つのみ
- excerptは100〜150字程度の概要文

---

## カバー画像のルール（必須）

新しく記事を作成する際は、`public/` フォルダ内の **未使用のUnsplash画像** の中から記事のテーマに合った画像を選ぶこと。

### 手順
1. `Glob` ツールで `public/*-unsplash.jpg` の一覧を取得する
2. `Grep` ツールで `content/posts/**/*.md` の `coverImage` フィールドを検索し、すでに使用済みの画像を特定する
3. **未使用の画像**の中から記事内容に最も関連する画像を選ぶ
4. フロントマターの `coverImage` に `"/<ファイル名>"` 形式で設定する

### 選定基準
- **記事の具体的なトピック・内容と視覚的に一致する画像を選ぶこと**（最重要）
  - 野球の記事には野球の画像、投資の記事には金融・チャートの画像、など
  - カテゴリーが同じでも記事の内容と無関係な画像は使用禁止
- 記事のカテゴリー・テーマと視覚的に一致する画像を優先する
- 同じ画像を複数の記事で使い回さない（既存記事と重複しないこと）
- 適切な未使用画像がない場合のみ、既使用画像の流用を許可する

### 画像選定の確認手順
画像を選んだ後、必ず以下を確認すること：
- 選んだ画像が記事の主題（スポーツ、テクノロジー、育児など）と一致しているか
- 記事を読んだ読者が「この画像は記事と関係ない」と感じないか
- 関連性が薄い場合は別の画像を選び直す
