# ブログサイト要件定義書

## プロジェクト概要

個人の技術ブログサイト。非技術者も記事を投稿できるMarkdownベースのブログシステム。

## 技術スタック

### フロントエンド
- **フレームワーク**: Next.js (App Router)
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS
- **コンテンツ管理**: Markdown (Git管理)
- **デプロイ**: Vercel
- **ドメイン**: カスタムドメイン使用

### 推奨ライブラリ
- `gray-matter`: Markdownのフロントマター解析
- `remark` / `remark-html`: Markdown → HTML変換
- `rehype-highlight` / `rehype-prism`: コードシンタックスハイライト
- `next-seo`: SEO対策
- `date-fns`: 日付フォーマット

## 機能要件

### 1. 記事一覧ページ
- [ ] 記事をカード形式で一覧表示
- [ ] 各カードに以下の情報を表示
  - タイトル
  - 公開日
  - カテゴリー
  - アイキャッチ画像（任意）
  - 概要（excerpt）
- [ ] ページネーション機能（1ページあたりの表示件数: 10-12件程度）
- [ ] 最新記事順に表示

### 2. 記事詳細ページ
- [ ] Markdownで記述された記事本文を表示
- [ ] 記事メタ情報の表示
  - タイトル
  - 公開日・更新日
  - カテゴリー
  - タグ（任意）
- [ ] シンタックスハイライト対応
- [ ] レスポンシブな画像表示
- [ ] SNSシェアボタン
  - Twitter (X)
  - Facebook
  - はてなブックマーク
  - その他（必要に応じて）

### 3. カテゴリー機能
- [ ] カテゴリー別記事一覧ページ
- [ ] カテゴリー一覧ページ（任意）
- [ ] ヘッダーまたはサイドバーにカテゴリーナビゲーション

### 4. 検索機能
- [ ] キーワードによる記事検索
- [ ] タイトル、本文、カテゴリー、タグを検索対象とする
- [ ] 検索結果一覧ページ
- [ ] クライアントサイド検索（静的サイト生成の場合）

### 5. 共通UI
- [ ] ヘッダー
  - サイトロゴ/タイトル
  - ナビゲーションメニュー
  - 検索ボックス
- [ ] フッター
  - コピーライト
  - SNSリンク
  - その他リンク
- [ ] レスポンシブ対応（モバイル、タブレット、デスクトップ）

## 非機能要件

- [ ] Lighthouse スコア 90点以上を目標
- [ ] 画像の最適化（next/image使用）
- [ ] 静的生成（SSG）を活用

### SEO
- [ ] メタタグの適切な設定
- [ ] OGP対応
- [ ] Twitter Card対応
- [ ] サイトマップ生成
- [ ] robots.txt設置

### アクセシビリティ
- [ ] セマンティックHTML
- [ ] キーボード操作対応
- [ ] 適切なコントラスト比

### 保守性
- [ ] 非技術者でもMarkdownファイルを編集できる構成
- [ ] Markdownファイルのテンプレート提供
- [ ] 記事投稿手順のドキュメント作成

## コンテンツ構成

### ディレクトリ構造（案）
```
/content
  /posts
    /2024
      /01
        article-slug.md
        another-article.md
      /02
        ...
/public
  /images
    /posts
      article-image.png
```

### Markdownフロントマター（案）
```yaml
---
title: "記事タイトル"
date: "2024-01-15"
updated: "2024-01-20"  # 任意
category: "カテゴリー名"
tags: ["tag1", "tag2"]  # 任意
excerpt: "記事の概要文"
coverImage: "/images/posts/cover.png"  # 任意
---
```
