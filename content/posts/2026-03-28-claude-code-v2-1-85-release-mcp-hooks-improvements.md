# Claude Code v2.1.85リリース！MCP環境変数・Hooksの条件フィルタなど実用的な改善が盛りだくさん

Claude Codeの最新バージョン **v2.1.85** がリリースされました。今回のアップデートでは、MCPサーバー連携の柔軟性向上、Hooks機能の条件フィルタ追加、スケジュールタスクのトランスクリプト改善など、日々の開発ワークフローをさらに快適にする変更が含まれています。

それぞれの新機能を、初心者の方にもわかりやすく解説していきます。

---

## 🔌 MCPサーバー向け環境変数の追加

### 何が変わった？

MCPサーバーとの連携時に使用する `headersHelper` スクリプトに、以下の2つの環境変数が新たに渡されるようになりました。

- **`CLAUDE_CODE_MCP_SERVER_NAME`** — 接続先MCPサーバーの名前
- **`CLAUDE_CODE_MCP_SERVER_URL`** — 接続先MCPサーバーのURL

### なぜ嬉しい？

これまで `headersHelper` スクリプトは、どのMCPサーバーに対してヘッダーを生成しているのかを知る手段がありませんでした。そのため、複数のMCPサーバーを使い分けている場合、サーバーごとに別々のヘルパースクリプトを用意する必要がありました。

今回の変更により、**1つのヘルパースクリプトで複数のMCPサーバーに対応**できるようになります。環境変数を参照して、サーバーごとに異なる認証トークンやカスタムヘッダーを返すことが可能です。

### 実用例

以下は、サーバー名に応じて異なるAPIキーをヘッダーとして返すヘルパースクリプトの例です。

```bash
#!/bin/bash
# headers_helper.sh — 複数MCPサーバーに対応するヘルパー

case "$CLAUDE_CODE_MCP_SERVER_NAME" in
  "production-server")
    echo '{"Authorization": "Bearer '"$PROD_API_KEY"'"}'
    ;;
  "staging-server")
    echo '{"Authorization": "Bearer '"$STAGING_API_KEY"'"}'
    ;;
  *)
    # デフォルトのヘッダー
    echo '{"Authorization": "Bearer '"$DEFAULT_API_KEY"'"}'
    ;;
esac
```

このスクリプトを1つ用意しておくだけで、本番・ステージングなど複数のMCPサーバーに対して適切な認証情報を自動で切り替えられます。管理がぐっとシンプルになりますね。

## 🎯 Hooksに条件フィルタ（`if` フィールド）が追加

### 何が変わった？

Hooks（フック）に新しい **`if`** フィールドが追加されました。このフィールドにはパーミッションルール構文（例：`Bash(git *)`）を指定でき、条件に一致する場合のみフックが実行されるようになります。

### なぜ嬉しい？

これまでのHooksは、対象イベントが発生するたびに毎回スクリプトが起動されていました。例えば「Bashコマンド実行時」にフックを設定すると、`ls` のような軽いコマンドでも毎回ヘルパープロセスが起動され、わずかながらオーバーヘッドが発生していました。

`if` フィールドを使うことで、**本当に必要な場面だけ**フックを実行できるようになり、プロセス生成のオーバーヘッドを大幅に削減できます。

### 具体的なイメージ

例えば、「gitコマンドが実行されたときだけログを取りたい」という場合、以下のように設定できます。

```json
{
  "hooks": {
    "on_command_execute": [
      {
        "if": "Bash(git *)",
        "command": "echo \"Git command detected\" >> ~/claude-git-log.txt"
      }
    ]
  }
}
```

この設定では、`git commit` や `git push` などのgit関連コマンドが実行されたときだけフックが発火します。`npm install` や `ls` などの無関係なコマンドではフックが動かないため、パフォーマンスへの影響を最小限に抑えられます。

## 📋 スケジュールタスクのトランスクリプトにタイムスタンプが追加

### 何が変わった？

`/loop` コマンドなどでスケジュールされたタスクのトランスクリプト（実行記録）に、**タイムスタンプマーカー**が追加されるようになりました。

### なぜ嬉しい？

スケジュールタスクは長時間にわたって繰り返し実行されることが多いため、「いつ何が起きたか」を後から追跡するのが難しい場面がありました。タイムスタンプが付くことで、トランスクリプトを見返す際に**時系列での把握**が格段にしやすくなります。

デバッグ時や、定期実行タスクの動作確認にとても役立つ改善です。
