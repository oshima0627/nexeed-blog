## Frontmatter（記事メタデータ）

記事の先頭には必ずFrontmatterを記載します。

```yaml
---
title: "【タイトル】サブタイトル｜キーワード"
date: "YYYY-MM-DD"
category: "カテゴリ名"
excerpt: "記事の要約（120〜150文字程度）"
affiliateBannerId: "バナーID"  # オプション：特定のバナーを指定する場合
---
```

### カテゴリ

以下の4つのカテゴリのいずれかを指定：

- `投資`
- `子育て`
- `ITエンジニア`
- `副業`

### タイトルの書き方

- **形式**: `【メインキーワード】具体的な内容｜ベネフィット`
- **例**: `【本音レビュー】スマイルゼミの評判・口コミ徹底調査｜利用者の体験談から見るメリット・デメリット`
- SEOを意識したキーワードを含める
- 読者のベネフィットを明確にする

## 記事の構成

### 基本構成

1. **H1タイトル**（Frontmatterのtitleと同じ）
2. **導入文**（2〜3段落）
   - 読者の悩みや課題を提起
   - 記事の概要を説明
3. **「この記事でわかること」セクション**
   - 箇条書きで記事の内容を列挙
4. **本文**（複数のH2セクション）
5. **まとめ**
6. **最後の一文**（行動を促す）
7. **締めの一文**（記事リンクの案内）

### 導入文の書き方

```markdown
「〇〇で悩んでいる…」「××したいけど△△で…」そんな悩みを抱えていませんか？

（課題の説明や背景情報）

本記事では、**〇〇について徹底解説**し、××のための具体的な方法をお伝えします。

**この記事でわかること:**

- ポイント1
- ポイント2
- ポイント3
```

### まとめセクション

```markdown
## まとめ

（記事の要点を3〜5つの段落でまとめる）

### 〇〇のポイントまとめ

- ポイント1
- ポイント2
- ポイント3

### 最後に

> （引用形式で最後のメッセージ）

（行動を促す一文）

---

**この記事が参考になった方は、ぜひ他の〇〇関連の記事もご覧ください！**
```

## Markdown書き方ルール

### 段落と改行

- 段落の間は1行空ける
- 文章内での改行は使用しない（自動折り返し）

### コードブロック

使用しない（技術記事以外）

## 太字（強調）の使い方

### 基本ルール

太字は `**テキスト**` で記述しますが、以下のルールに従ってください。

### ❌ 避けるべきパターン

#### 1. 括弧を太字内に含めない

```markdown
❌ 悪い例: **「丸つけ不要＆学習管理が簡単」**
✅ 良い例: 「**丸つけ不要＆学習管理が簡単**」

❌ 悪い例: **（12ヶ月時点）**
✅ 良い例: （**12ヶ月時点**）

❌ 悪い例: **"テキスト"**
✅ 良い例: "**テキスト**"

❌ 悪い例: **(補足)**
✅ 良い例: (**補足**)
```

**理由**: Markdownパーサーが正しく太字を認識しない場合があるため、括弧は太字の外に出す。

#### 2. 括弧と後続テキストが混在する場合

```markdown
❌ 悪い例: **「今日やること」が明確**
✅ 良い例: 「**今日やること**」**が明確**
```

**理由**: 括弧内と括弧外のテキストを別々に太字にすることで、確実に太字が適用される。

#### 3. 末尾に括弧がある場合

```markdown
❌ 悪い例: **継続率（12ヶ月時点）**
✅ 良い例: **継続率**（12ヶ月時点）
```

**理由**: 補足的な情報（括弧内）は太字にする必要がない場合が多い。

#### 4. パーセント記号を含む場合

```markdown
❌ 悪い例: **学習習慣定着率は84%**
✅ 良い例: **学習習慣定着率は84**%

❌ 悪い例: **導入率：55.1％**
✅ 良い例: **導入率：55.1**％
```

**理由**: パーセント記号は単位であり、強調の対象ではないため太字の外に出す。

### ✅ 使用すべきパターン

#### ラベルやヘッダー

```markdown
**基本情報:**
**メリット:**
**注意点:**
```

コロン（`:`）で終わるラベルは太字を使用してOK。

#### 重要なキーワード

```markdown
本記事では、**TwitterやブログなどのSNS、保護者向けコミュニティに投稿された利用者の口コミ・評判を徹底調査**し、そのリアルな声をまとめました。
```

文中の重要なフレーズを強調する場合に使用。

#### 表内のセル

```markdown
| 項目 | 内容 |
|------|------|
| **合計** | **34,100円** |
```

表内で特定のセルを強調する場合に使用。

## 統計データの書き方

### 数値データの記載方法

```markdown
### データタイトル

〇〇省の調査によると、××は**YY**%という結果が出ています。

**〇〇の推移:**

| 年度 | 数値 | 前年比 |
|------|------|--------|
| 2020年 | 150億円 | +20% |
| 2021年 | 180億円 | +20% |
| 2022年 | 220億円 | +22% |

出典: 〇〇省「調査名」（YYYY年）
```

### 出典の記載

- データの直後に必ず出典を記載
- 形式: `出典: 組織名「調査名」（YYYY年）`
- 架空のデータを使用する場合でも出典形式を守る

### パーセント記号のルール

```markdown
✅ 良い例: **導入率は56**%
✅ 良い例: **55.1**％
❌ 悪い例: **56%**
❌ 悪い例: **55.1％**
```

## 引用の書き方

### 口コミ・評判の引用

```markdown
> 「〇〇〇〇〇〇〇〇〇〇〇〇」（情報源）
```

**例:**
```markdown
> 「小2の娘が毎日自分からタブレットを開いて勉強している。紙の教材は溜まる一方だったのに、スマイルゼミは続いている。ごほうびゲームが楽しみらしい」（保護者ブログより）
```

### まとめセクションの引用

```markdown
> （メッセージ）
```

**例:**
```markdown
> 「老後のお金が心配…」という不安を抱えたまま過ごすより、今から正しい知識を身につけて、計画的に準備する方が、はるかに安心です。
```

## 表（テーブル）の書き方

### 基本形式

```markdown
| 項目 | 内容 |
|------|------|
| 行1列1 | 行1列2 |
| 行2列1 | 行2列2 |
```

### 数値データの表

```markdown
| 年度 | 市場規模 | 前年比 |
|------|---------|--------|
| 2020年 | 150億円 | +20% |
| 2021年 | 180億円 | +20% |
| 2022年 | 220億円 | +22% |
```

### 合計行の強調

```markdown
| 項目 | 料金 |
```

## 見出しの書き方

### 見出しレベル

```markdown
# H1: 記事タイトル（1つのみ）

## H2: 大項目

### H3: 中項目

#### H4: 小項目
```

### 見出しのルール

- H1は記事タイトルのみ（1つだけ）
- H2から順番に使用（H3の前にH2が必要）
- 見出しには質問形や「〜について」などの表現を使う

**例:**
```markdown
## スマイルゼミとは？

### サービスの概要

### なぜ人気なのか？

#### 理由1: 自動採点機能

#### 理由2: ごほうびシステム
```

## アフィリエイトバナーの設定

### Frontmatterでの指定

特定のアフィリエイトバナーを表示したい場合、Frontmatterに `affiliateBannerId` を指定します。

```yaml
---
title: "記事タイトル"
date: "2026-01-19"
category: "子育て"
excerpt: "記事の要約"
affiliateBannerId: "kidsrobo-300x250"
---
```

### バナーIDの確認

バナーIDは `data/affiliate-links.ts` ファイルで定義されています。

詳細は以下のドキュメントを参照：
- [AFFILIATE_USAGE.md](./AFFILIATE_USAGE.md) - もしもアフィリエイトの使い方
- [A8_AFFILIATE_USAGE.md](./A8_AFFILIATE_USAGE.md) - A8.netアフィリエイトの使い方

## 最新情報の使用

### 基本ルール

記事を作成する際は、必ず最新の情報を使用してください。

### 情報収集の原則

1. **公式サイトからの情報収集**
   - サービスや製品に関する記事を書く場合、必ず公式サイトから最新情報を取得する
   - 公式サイトのニュース、プレスリリース、サービス紹介ページを確認する
   - 公式ブログやお知らせページをチェックする

2. **情報の鮮度確認**
   - 統計データや調査結果は、必ず最新年度のものを使用する
   - 古い情報（1年以上前）は原則として使用しない
   - データの出典に年度を明記する

3. **料金・プラン情報**
   - 料金やプラン内容は変更される可能性が高いため、必ず公式サイトで最新情報を確認する
   - 記事作成日時点での情報であることを明記する
   - 「料金は変更される可能性があります。最新情報は公式サイトをご確認ください」などの注意書きを追加する

4. **サービス内容・機能**
   - サービスの機能や特徴は、公式サイトの最新情報に基づいて記載する
   - 口コミや評判を引用する場合も、できるだけ新しいものを選ぶ
   - 古い情報や噂、未確認情報は使用しない

### 情報源の明記

```markdown
**本記事は2026年1月時点の情報に基づいています。最新情報は公式サイトをご確認ください。**

出典: PyQ公式サイト（2026年1月確認）
```

### 禁止事項

- ❌ 古い情報をそのまま使用する
- ❌ 公式サイトを確認せずに記憶や推測で記事を書く
- ❌ 出典不明の情報を使用する
- ❌ 他のブログやメディアの情報を丸写しする

## 記事作成チェックリスト

記事を作成したら、以下の項目をチェックしてください。

### Frontmatter

- [ ] titleが適切なフォーマットになっている
- [ ] dateが正しい
- [ ] categoryが4つのカテゴリのいずれかである
- [ ] excerptが120〜150文字程度である

### 構成

- [ ] H1タイトルがある（1つのみ）
- [ ] 導入文がある（読者の悩みを提起）
- [ ] 「この記事でわかること」セクションがある
- [ ] まとめセクションがある
- [ ] 最後に締めの一文がある

### Markdown

- [ ] 太字の括弧が正しい（括弧は太字の外）
- [ ] パーセント記号が太字の外に出ている
- [ ] 統計データに出典が記載されている
- [ ] 引用が正しい形式（`>`）で記載されている
- [ ] 表が正しい形式で記載されている

### 内容

- [ ] 読者にとって有益な情報が含まれている
- [ ] 具体的なデータや事例が含まれている
- [ ] 行動を促す一文がある

### 最新情報

- [ ] 公式サイトから最新情報を取得している
- [ ] 統計データは最新年度のものを使用している
- [ ] 料金・プラン情報は現在のものである
- [ ] 情報の出典と確認日を明記している

## よくある間違い

### 1. 太字の括弧

```markdown
❌ **「テキスト」**
✅ 「**テキスト**」
```

### 2. パーセント記号

```markdown
❌ **84%**
✅ **84**%
```

### 3. 出典の記載漏れ

```markdown
❌ 〇〇省の調査によると、××は56%です。

✅ 〇〇省の調査によると、××は**56**%です。

出典: 〇〇省「調査名」（2024年）
```

### 4. 見出しの順序

```markdown
❌
## H2
#### H4  ← H3がないのにH4を使用

✅
## H2
### H3
#### H4
```
